# 项目可用 SKILLS 参考指南

> 基于当前项目技术栈：Next.js 15 + React 19 + TypeScript + Tailwind CSS v4 + pnpm monorepo + Electron + Zustand + TanStack Query

---

| Skill | Summary | Description |
|-------|---------|-------------|
| `next-best-practices` | Next.js 最佳实践 | 文件约定、RSC 边界、数据模式、async API、元数据、错误处理、路由处理器、图片/字体优化、打包 |
| `next-cache-components` | Next.js 16 缓存组件 | PPR、use cache 指令、cacheLife、cacheTag、updateTag 缓存策略 |
| `frontend-design` | 前端 UI 设计 | 创建高质量前端界面、组件、页面，避免通用 AI 美学，生成独特生产级代码 |
| `clean-code` | 整洁代码规范 | 命名约定、函数拆分、错误处理、单一职责、注释纪律、代码审查 |
| `clean-architecture` | 整洁架构 | 依赖规则、架构分层、端口与适配器、用例边界、框架独立 |
| `refactoring-patterns` | 重构模式 | 提取方法、替换条件、技术债务处理、安全重构序列 |
| `software-design-philosophy` | 软件设计哲学 | 模块设计、信息隐藏、深度模块、复杂度管理、API 设计 |
| `database-schema-designer` | 数据库设计 | SQL/NoSQL 模式设计、规范化、索引策略、迁移、性能优化 |
| `coding-guidelines` | Rust 代码规范 | 命名、格式化、注释、Clippy、rustfmt、最佳实践（如项目涉及 Rust） |
| `tdd` | 测试驱动开发 | Red-Green-Refactor 循环、集成测试、测试优先开发 |
| `prototype` | 快速原型 | 创建可运行的终端应用或 UI 变体，探索设计方案 |
| `wireframe-prototyping` | 线框原型 | 低保真视觉原型，规划信息架构、用户流、界面设计 |
| `brainstorming` | 头脑风暴 | 创建功能/组件/修改前的意图探索、需求和设计讨论 |
| `writing-plans` | 编写计划 | 多步任务规范和执行计划，在编码前对齐方案 |
| `prd` | 产品需求文档 | 创建结构化任务列表、用户故事、验收标准 |
| `to-issues` | 转为 Issue | 将计划/规范拆分为独立的 Issue，使用垂直切片 |
| `to-prd` | 生成 PRD | 将当前对话上下文转化为 PRD 并发布到 Issue 跟踪器 |
| `git-commit` | Git 提交 | 约定式提交、智能暂存、提交消息生成 |
| `review` | 代码审查 | 审查代码变更的正确性、安全性、质量、性能 |
| `diagnose` | 故障诊断 | 复杂 Bug 和性能回归的结构化诊断循环 |
| `go-code-review` | Go 代码审查 | Go 代码审查和社区风格标准（如项目涉及 Go 后端） |
| `tailwind-design` | Tailwind 设计系统 | Tailwind CSS 设计令牌、主题、组件样式（项目使用 Tailwind v4） |
| `component-library` | 组件库开发 | 设计系统、组件库架构、API 设计、文档（对应 packages/admin-system） |
| `monorepo-management` | Monorepo 管理 | pnpm workspace、Turborepo 配置、包依赖管理、构建优化 |
| `electron-packaging` | Electron 打包 | Electron 开发、打包、跨平台构建（对应 apps/admin 的 Electron 支持） |
| `responsive-design` | 响应式设计 | 移动端优先、断点设计、触摸优化、移动适配 |
| `accessibility-audit` | 无障碍审计 | WCAG 合规、ARIA 属性、键盘导航、屏幕阅读器支持 |
| `performance-optimization` | 性能优化 | Core Web Vitals、代码分割、懒加载、Bundle 分析 |
| `state-management` | 状态管理 | Zustand、Context、TanStack Query 状态管理模式（项目使用 Zustand + React Query） |
| `form-handling` | 表单处理 | React Hook Form + Zod 验证模式（项目使用 RHF + Zod） |
| `api-integration` | API 集成 | REST/GraphQL 客户端、错误处理、缓存策略、类型安全 |
| `testing-strategy` | 测试策略 | 单元测试、集成测试、E2E 测试、Mock 策略 |
| `ci-cd` | CI/CD | 持续集成/部署、GitHub Actions、自动化测试、部署流水线 |
| `documentation` | 文档编写 | README、API 文档、架构决策记录、CHANGELOG |
| `i18n-l10n` | 国际化 | 多语言支持、本地化工作流、翻译管理 |
| `seo-optimization` | SEO 优化 | Next.js SEO、元数据、结构化数据、站点地图 |
| `security-audit` | 安全审计 | XSS、CSRF、认证授权、依赖漏洞、安全最佳实践 |
